<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP/remote -> set_agent_policy | An MCP caller should not be able to escalate permissions or raise the agent cap through a tool call |
| MCP/remote -> secret fields in policy params | Secret-named keys in policy params must be rejected |
| Settings read -> router constructor | Persisted agentCap is read at startup; a malicious profile could set a very high cap |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02 | Elevation of Privilege | routeSetAgentPolicy | mitigate | Validate agentCap range 1..16; reject secret/apiKey/password/token keys with PERMISSION_BLOCKED; require bearer auth (inherited from routeForSession) |
| T-08-02b | Information Disclosure | routeGetAgentPolicy | mitigate | Policy response must not include raw secret values; return only settings keys and boolean/int values |
| T-08-02c | Tampering | AgentRuntime finishTask | mitigate | executionMode field is set by trusted runtime code, not by MCP caller-supplied metrics; MCP submit_task does not control execution-mode outcome |
| T-08-02d | Spoofing | Settings agentCap read at startup | accept | Cap is clamped to 1..16 at read time; environment variable fallback is a bootstrap override only; threat is low for local single-user deployment |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No new package manager installs; in-tree C++ Qt code only |
</threat_model>
