<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| toolbar/menu actions -> popup | User input creates native popup windows tied to current BrowserWindow models. |
| popup activation -> tab focus | Keyboard or mouse activation switches tabs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-06-01 | Spoofing | Toolbar buttons | mitigate | Use distinct labels/tooltips and existing icon patterns; do not reuse secure-origin or automation badge colors for neutral controls. |
| T-07-06-02 | Denial of Service | Popup lifetime | mitigate | Parent popups to BrowserWindow/NavigationBar and close on Esc/window destruction. |
| T-07-06-03 | Tampering | Quick switch activation | mitigate | Activate tabs through existing `WebTab::makeCurrentTab` or `TabWidget::setCurrentIndex`, not direct widget mutation. |
</threat_model>
