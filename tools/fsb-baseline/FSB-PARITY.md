# FSB Parity Evidence Ledger

This file is the authoritative parity matrix for Prometheus against FSB (Full Self Browsing).
Each row tracks one FSB feature category, the Prometheus native equivalent, implementation
status, any native improvement over FSB, and the validation command or evidence path.

**Purpose:** FSBP-02 requires a maintained matrix covering all required FSB categories.
FSBP-03 requires native improvements to be documented and verified. FSBP-06 requires
release validation to fail when rows are missing, regressed, undocumented, or unverified.

**Adding a row:** Add one row per new FSB category. Status must be one of exactly:
`Missing`, `Partial`, `Parity`, `Better`, or `Verified`. `Better` requires both the
Native Improvement and Validation columns to be non-empty. `Verified` requires a
passing validation command or evidence file path.

**Do not place raw secret values, literal API keys, passwords, or card numbers in this file.**

## Notes

- **Better** status requires both the Native Improvement column and the Validation column to be non-empty.
- **Verified** status requires an evidence file path and a passing validation command.
- Rows marked **Partial** indicate categories where the Prometheus implementation exists but does not yet fully match the FSB feature set; validation commands still apply to the implemented subset.
- This file must not contain raw secret values, literal API keys, passwords, or card numbers.
- The parity smoke gate (`smoke-fsb-plus-parity.sh`) checks for required row categories and source patterns at release time; see FSBP-06 above.
