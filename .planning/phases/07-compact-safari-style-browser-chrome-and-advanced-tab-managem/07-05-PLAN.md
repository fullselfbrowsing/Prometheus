<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page metadata -> tab search UI | Titles and URLs from web content render in a privileged native popup. |
| keyboard/mouse input -> tab activation/actions | User input switches tabs or opens row actions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05-01 | Information Disclosure | `TabSearchDelegate` | mitigate | Render existing title/URL/domain only; do not render hidden page text, form content, request params, or secrets. |
| T-07-05-02 | Denial of Service | `TabSearchFilterModel` | mitigate | Use simplified bounded filtering and fixed row geometry so long titles/URLs cannot resize rows or hang filtering. |
| T-07-05-03 | Spoofing | URL/domain display | mitigate | Middle-elide URL/domain and keep favicon/security cues distinct from automation badges. |
| T-07-05-04 | Tampering | Quick switch filtering | mitigate | Filtering over `TabMruModel` must not mutate MRU source order; activation uses existing tab switching. |
</threat_model>
