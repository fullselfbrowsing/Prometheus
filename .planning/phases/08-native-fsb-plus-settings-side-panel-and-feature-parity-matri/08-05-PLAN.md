<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Native UI -> AgentRuntime::saveVaultSecret | Vault secret save path is native-only (allowSecretPayload=true); MCP must not be able to trigger this |
| Native UI -> AgentRuntime::saveProviderConfig | Provider key save is native-only; must clear field after save |
| Supervision pairing display -> router diagnostics | Session fields displayed must not expose the actual pairing secret key value |
| Parity Matrix section -> QProcess smoke script | Running a shell subprocess from UI; must use fixed path not user-controlled input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Information Disclosure | Vault metadata list | mitigate | List rows show only origin, label, secret-state chip, and autofill policy; saveVaultSecret caller sets allowSecretPayload=true; list never reads back raw secret |
| T-08-05b | Information Disclosure | Provider key display | mitigate | Key field is password echo; after save the field is cleared; list rows show only Key saved/No key saved label |
| T-08-05c | Elevation of Privilege | Permissions & Agents section saving cap | mitigate | set_agent_policy validates range 1..16 and rejects secret-named keys (Plan 02); control panel calls routeForSession(request, "control_panel"), not a direct member mutation |
| T-08-05d | Tampering | QProcess smoke script invocation | mitigate | Script path is a fixed SOURCE_ROOT constant; no user-supplied path or shell injection possible; QProcess uses QStringList args, not a shell command string |
| T-08-05e | Spoofing | Supervision session display | mitigate | Session fields sourced from router diagnostics(), not from user input; expired/stale sessions remain visible until explicitly dismissed |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No new package manager installs; in-tree C++ Qt code only |
</threat_model>
