<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| human UI -> tab/group actions | Destructive and ownership-affecting actions are triggered from menus/popups. |
| MCP/native reads -> external agents | Group and tab state leaves browser internals through read APIs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-08-01 | Tampering | Group/action menus | mitigate | Route actions through existing TabWidget methods and Plan 07-07 group operations; do not mutate WebTab arrays directly from UI rows. |
| T-07-08-02 | Elevation of Privilege | MCP group operations | mitigate | Keep group exposure read-only unless mutation is explicitly routed through ownership checks; unsupported mutation tools return typed errors. |
| T-07-08-03 | Repudiation | Destructive close/unload/detach | mitigate | Use existing confirmation paths and keep actions routed through existing tab methods that preserve closed-tab recovery. |
| T-07-08-04 | Information Disclosure | `list_tabs` group/state fields | mitigate | Expose only local tab metadata and sanitized state fields; no page text, secrets, prompts, or vault/provider data. |
</threat_model>
