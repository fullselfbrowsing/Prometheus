# Phase 07 — UI Design Contract

> Visual and interaction contract for compact native browser chrome, advanced tab management, and agent-aware tab state. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Native Qt Widgets / existing Falkon chrome widgets |
| Icon library | Existing `QIcon::fromTheme` plus bundled Prometheus fallback assets; no new third-party registry assets in this phase |
| Font | Platform system UI font via Qt defaults |

Source notes: This phase is a C++/QtWebEngine desktop browser phase, not React/Next/Vite. `components.json` and Tailwind config are absent, so the shadcn gate is not applicable. The visual baseline comes from Phase 2: Prometheus near-black surfaces, FSB orange accents, and compact native chrome.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tab badge offsets, field inset adjustments |
| sm | 8px | Compact element spacing, row gaps, menu padding |
| md | 16px | Default panel padding, favicon/icon size |
| lg | 24px | Section padding, compact toolbar control minimum visual span |
| xl | 32px | Compact toolbar height, tab row height, icon-only button target |
| 2xl | 48px | Tab overview group spacing, large popover padding |
| 3xl | 64px | Wide overview gutters and major layout breaks |

Exceptions: use 36px only as the maximum compact toolbar button width when preserving existing Falkon `ToolButton` proportions; use 40px only as the minimum active compact-address height in high-DPI themes. All new spacing, margins, and gaps must use the declared 4px scale.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.5 |
| Label | 12px | 600 | 1.3 |
| Heading | 16px | 600 | 1.2 |
| Display | 20px | 600 | 1.2 |

Rules: use only weights 400 and 600. Do not use negative letter spacing or viewport-scaled type. Compact tab titles must elide right; URLs in overview/search must elide middle so domains remain readable.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#090909` | Main window background, inactive chrome base, page-adjacent browser surface |
| Secondary (30%) | `#141414` | Toolbar/tab surfaces, tab overview panel, sidebar-adjacent tab group surfaces |
| Accent (10%) | `#ff6b35` | Active compact address/tab focus ring, selected tab group marker, primary tab-management CTA, active automation/supervision indicator |
| Destructive | `#ff6666` | Destructive actions only: close group, close other tabs, clear closed-tabs list, failed tab-health state |

Accent reserved for: active tab/address focus, selected tab group underline/rail, primary CTA `New Tab Group`, active agent automation badge, supervised-session badge, focused keyboard ring, and hover/focus glow with existing secondary orange `#ff8c42`.

Do not use orange for ordinary inactive tabs, secondary buttons, neutral hover states, secure-origin cues, or decorative fills. Secure-origin cues must stay tied to the existing site/security icon treatment so they are not confused with automation state.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | New Tab Group |
| Empty state heading | No tabs found |
| Empty state body | Clear the search or open a new tab group. |
| Error state | Tab action failed. Refresh the tab list and try again. |
| Destructive confirmation | Close Tab Group: Close all tabs in this group? This can be undone from Recently Closed Tabs.; Close Other Tabs: Close all other tabs?; Close Tabs to the Right/Left: Close these tabs?; Clear Closed Tabs: Clear the recently closed tabs list? |

Copy rules: use short native-browser verbs. Prefer `Tab Overview`, `Search Tabs`, `New Tab Group`, `Compact`, `Separate`, `Title and Icon`, and `Favicon Only`. Do not describe how the UI works in permanent on-screen help text; use tooltips, menu labels, and accessible names.

## Phase-Specific Visual Contract

### Compact Chrome

- Compact mode must use one primary chrome row. Back, forward, reload/stop, tab strip, active tab/address field, tab overview/search, Prometheus agent/sidebar entry, and tools menu must fit without overlap.
- The active tab is the unified address/search field. It contains favicon/security icon, editable address/search text, load progress, and active tab state badges.
- Inactive tabs sit in the same toolbar row. Default tab display is title plus favicon. Favicon-only mode must be available and must not change row height.
- Separate layout remains available and preserves the existing conventional toolbar plus tab bar behavior.
- Compact toolbar height is 32px content inside a 40px maximum row. Icon-only controls use 16px icons in 32px targets.
- Active compact tab/address width: preferred 320px, minimum 180px, maximum 520px. Inactive tab width: preferred 144px, minimum 72px, maximum 176px. Pinned/favicon-only tabs: 32px.
- When width is constrained, collapse inactive tabs from title-plus-icon to favicon-only before shrinking the active address field below 180px. If overflow remains, show the existing open-tabs overflow affordance plus `Search Tabs`.

### Tab State

- Agent ownership appears as a small state badge on the right edge of a tab. Use a 12px label only when space allows; otherwise use an 8px dot with tooltip and accessible name.
- Active automation uses the reserved accent color and must be visible on both compact and separate layouts.
- Supervision state uses the reserved accent badge plus tooltip `Supervised session active`.
- Tab health warning uses destructive red only when the tab has failed, is stale, or cannot be controlled.
- Loading, audio, muted, pinned, restored/unloaded, private, and secure-origin cues must remain distinct from agent/supervision cues.

### Tab Groups

- Groups appear as native tab metadata, not decorative cards. In compact chrome, group membership is shown by a 2px top rail or underline and a group label in overview/search.
- Group label height is 24px, label text is 12px semibold, and group row spacing is 8px.
- Group swatches must use muted non-dominant colors and never replace the FSB orange accent contract. Default swatches: `#3a7bd5`, `#2f9e7e`, `#9b6ade`, `#d29b36`, `#a65b5b`.
- Collapsed groups show one group chip plus the active or most recent favicon. Expanded groups show member tabs in current order.

## Interaction Contract

| Surface | Contract |
|---------|----------|
| Layout setting | Add `Tab Layout` with exactly two values: `Compact` and `Separate`. Persist under browser tab settings. |
| Tab display setting | Add `Tab Display` with exactly two values: `Title and Icon` and `Favicon Only`. Row height must not change when toggled. |
| Active address/search | `Ctrl+L`, `Alt+D`, and File > Open Location focus the active compact tab/address field. Completion behavior stays identical to existing `LocationBar`. |
| New tab | `Ctrl+T`, toolbar add-tab, and middle-click add-tab behavior stay compatible with current Falkon behavior. |
| Close/restore | `Ctrl+W` closes the current tab; `Ctrl+Shift+T` restores a closed tab. Closing multiple tabs or a group requires confirmation. |
| Reorder | Dragging tabs reorders within the current group; dragging across group boundaries moves the tab into the target group. |
| Duplicate/detach/unload | Keep existing context-menu actions for Duplicate Tab, Detach Tab, Load Tab, and Unload Tab. Add the same actions to overview/search row context menus. |
| Search tabs | `Search Tabs` opens a dense native popover with search input, MRU ordering option, group filters, and row actions. Empty state uses the copy above. |
| Tab overview | `Tab Overview` opens a native dense overview organized by group, not a marketing-style card grid. Rows are 32px tall with favicon, title, URL/domain, owner/supervision/health badges, and hover actions. |
| Quick switch | Quick switch uses the existing `TabMruModel` ordering. The popover must support keyboard navigation, Enter to switch, Esc to close, and typed filtering. |
| Agent boundaries | Agent-owned tabs can be viewed by the human, but agent ownership and active automation must remain visible before a user closes, unloads, detaches, or moves the tab. |

Mouse and trackpad behavior must remain Mac-native and compatible with existing Falkon tab behavior: middle-click close where already supported, wheel switching only when the existing setting allows it, drag/drop reorder, and context menus on secondary click.

## Responsive And Verification Contract

| Window Width | Required Behavior |
|--------------|-------------------|
| Narrow desktop, 800px | No overlapping text or icons. Inactive tabs may collapse to favicon-only. Active address/search remains at least 180px. Search Tabs remains reachable. |
| Standard desktop, 1280px | Compact row shows navigation controls, active address/search, 5-7 ordinary tabs with readable titles, and agent/supervision indicators. |
| Wide desktop, 1728px+ | Compact row uses available space without oversized controls. Inactive tab max width stays 176px and active address/search max width stays 520px. |

Visual checks must cover compact/separate layout, title-and-icon/favicon-only display, active automation, supervised tab, warning tab, private window, grouped tabs, overflow tabs, tab overview, and search empty/error states.
