# Phase 10 — UI Design Contract: New Tab Start Page

> Visual and interaction contract for the Prometheus start page surface.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> This is a native Qt6/C++ surface. The JSX prototype in
> `.planning/design/prometheus-browser/prometheus/StartPage.jsx` is the
> look-and-behavior target only — it is never shipped. All tokens, fonts,
> icons, and the PrometheusMarkWidget are Phase 9 deliverables reused here.

---

## Implementation Mechanism

### How New Tab is served today

Falkon's `TabWidget` reads `newTabUrl` from settings (default
`falkon:speeddial`). When a new tab opens, it loads that URL. The
`FalkonSchemeHandler` handles `falkon:` requests; `FalkonSchemeReply::startPage()`
already serves `falkon:start` by reading `:html/start.html` from Qt resources and
doing token substitution.

### Phase 10 replacement strategy

Replace the existing `falkon:start` HTML resource with the Prometheus start page.
Change the default `newTabUrl` from `falkon:speeddial` to `falkon:start` in
`TabWidget` settings. The `FalkonSchemeReply::startPage()` method replaces the
current placeholder HTML with the full Prometheus start page HTML.

The start page is a self-contained HTML/CSS document embedded in Qt resources at
`:html/start.html`. It uses `qrc:` URLs for fonts (Poppins, Space Mono) and Font
Awesome icons already bundled in Phase 9. The `FalkonSchemeHandler` already has a
`QWebChannel` wired per `WebPage` — the start page uses this channel to call into
native C++ for the Ask FSB handoff.

### Ask FSB handoff contract (START-02, START-04)

The start page web content communicates with the native sidebar via the existing
`QWebChannel` that `WebPage` installs on every page (`ExternalJsObject::setupWebChannel`
registers `qz_object`). Phase 10 registers an additional `QWebChannel` object for
the start page:

**Native side — new C++ class `PrometheusStartBridge` (QObject):**

```cpp
class FALKON_EXPORT PrometheusStartBridge : public QObject
{
    Q_OBJECT
public:
    explicit PrometheusStartBridge(BrowserWindow* window, QObject* parent = nullptr);

public Q_SLOTS:
    // Called from JS: seeds prompt into sidebar FSB Agent composer and opens panel.
    void openAgentWithPrompt(const QString& prompt);

    // Called from JS: navigate to a URL in the current tab.
    void navigateTo(const QString& url);

    // Called from JS: returns JSON array of bookmark URLs for favorites grid.
    QString bookmarkFavoritesJson() const;
};
```

Registration: In `ExternalJsObject::setupWebChannel`, detect when the page URL is
`falkon:start` and also register a `PrometheusStartBridge` instance as
`qz_prometheus_start`. The start page HTML calls:

```js
new QWebChannel(qt.webChannelTransport, function(channel) {
    var bridge = channel.objects.qz_prometheus_start;
    // Ask FSB button / Enter key / suggestion card click:
    bridge.openAgentWithPrompt(promptText);
    // Favorites tile click:
    bridge.navigateTo(url);
});
```

**`openAgentWithPrompt` implementation steps (native):**

1. Set `AgentRuntimeSidebarWidget::m_taskPrompt` text to `prompt`.
2. Call `BrowserWindow::sideBarManager()->showSideBar("PrometheusAgent", false)` —
   the `false` argument means "show, do not toggle off if already open".
3. Switch `m_modeStack` to index 0 (FSB Agent mode) if not already active.

To expose `m_taskPrompt` and mode-switch capability, `AgentRuntimeSidebarWidget`
needs one new public slot:

```cpp
// In agentruntimesidebar.h
public Q_SLOTS:
    void seedPromptAndOpen(const QString& prompt);
```

`seedPromptAndOpen` sets `m_taskPrompt->setPlainText(prompt)` and calls
`m_modeStack->setCurrentIndex(0)`. The `PrometheusStartBridge` then calls this
slot on the widget after retrieving it from the sidebar.

**Why not a `falkon:` scheme redirect or URL-query approach:**
A `falkon:` redirect cannot pass a freeform prompt string safely across a navigation.
A URL-query approach would expose the prompt in the address bar and break on
multi-line prompts. The QWebChannel path is already present, is synchronous from
the page's perspective, does not require network I/O, and cannot leak origin data
across tabs.

### Favorites data source (START-03)

`PrometheusStartBridge::bookmarkFavoritesJson()` queries `mApp->bookmarks()->rootItem()`
and returns the first 8 URL bookmarks as a JSON array:

```json
[
  { "id": "bm-1", "name": "GitHub", "url": "https://github.com", "color": "#24292f", "mono": "GH" },
  ...
]
```

`color` is the bookmark's stored color if present, otherwise a deterministic warm
accent shade derived from the URL host. `mono` is the first two characters of the
domain name, uppercase. The start page JS calls `bookmarkFavoritesJson()` on
`DOMContentLoaded` and renders the grid from the response. If no bookmarks exist,
the grid shows the empty state (see Copywriting Contract).

### Suggested automations data source (START-04)

Static curated set, hardcoded in the start page HTML as a JS constant (same as the
JSX prototype `FSB_SUGGESTIONS`). No backend read required. Per `10-CONTEXT.md`
Deferred Ideas, dynamic/personalized suggestions are out of scope for Phase 10.

## Interaction Contract

### Keyboard

| Key | Context | Action |
|-----|---------|--------|
| Tab (page open) | On load | Focus lands on omnibox input automatically |
| `Enter` | Omnibox focused, input non-empty | Evaluate input: if it is a URL or plain domain (`https://…`, `domain.tld`), navigate the current tab. Otherwise, call `openAgentWithPrompt(input)` — same as Ask FSB button |
| `Escape` | Omnibox focused | Clear omnibox input |
| `Enter` | Ask FSB button focused | Trigger Ask FSB (same as click) |
| `Space` / `Enter` | Favorites tile focused | Navigate to tile URL |
| `Space` / `Enter` | Suggestion card focused | Call `openAgentWithPrompt(card.prompt)` |

Note: "URL detection" heuristic: input matches `/^https?:\/\//` or contains a `.`
with no spaces and at least 2 characters after the dot. All other input is treated
as an agent prompt.

### Ask FSB handoff sequence

1. User types a natural language prompt in the omnibox and presses Enter, or clicks
   "Ask FSB", or clicks a suggestion card.
2. Start page JS calls `bridge.openAgentWithPrompt(prompt)` via QWebChannel.
3. `PrometheusStartBridge::openAgentWithPrompt` on the C++ side:
   a. Retrieves the active `AgentRuntimeSidebarWidget` from the window's sidebar.
   b. Calls `widget->seedPromptAndOpen(prompt)` — sets `m_taskPrompt` text and
      switches mode stack to index 0 (FSB Agent).
   c. Calls `window->sideBarManager()->showSideBar("PrometheusAgent", false)`.
4. The side panel animates open (if not already open) to the FSB Agent tab.
5. The user sees their prompt pre-filled in the composer and can edit before running,
   or press Ctrl+Return to run immediately.

The prompt is never auto-submitted. The user must explicitly run the task. This
preserves the human-in-the-loop supervision model.

### Favorites click

1. User clicks a favorites tile.
2. JS calls `bridge.navigateTo(url)`.
3. `PrometheusStartBridge::navigateTo` calls `m_window->weView()->load(QUrl(url))`.
4. Current tab navigates to the URL (start page is replaced in-tab).

## Empty and Error States

### Empty favorites grid

Displayed when `bookmarkFavoritesJson()` returns an empty array.

```
[ fa-star icon, --text-3, 24px ]
No favorites yet
Open a page and bookmark it — it will appear here.
```

Layout: centered in the `.pm-fav-wrap` area, same width constraint. Icon size 24px
(`--text-3`), heading 14px Poppins 600 (`--text`), body 13px system UI (`--text-3`).

### Bridge unavailable / QWebChannel timeout

If the `qz_prometheus_start` object is not available within 2 seconds of page load,
the omnibox Ask FSB button is hidden and replaced with a text notice:

```
[ fa-triangle-exclamation, --text-3, 14px ]
Agent bridge unavailable — reload the tab to restore.
```

Plain text link "Reload" (`color: var(--accent)`) triggers `window.location.reload()`.

### Private browsing overlay

When `mApp->isPrivate()` is true, the `FalkonSchemeReply::startPage()` method
already injects `%PRIVATE-BROWSING%`. Keep this behavior; the private notice renders
above the hero mark.

## Theme Switching

The start page HTML applies `data-theme` on `<html>`. The `PrometheusStartBridge`
exposes:

```cpp
Q_INVOKABLE QString currentTheme() const;
// Returns "dark" or "light"
```

The JS reads `bridge.currentTheme()` on `DOMContentLoaded` and sets
`document.documentElement.setAttribute('data-theme', theme)`. The browser shell also
emits `loadPrometheusVariant` changes; the start page does not need to respond to
live theme changes (a reload on theme switch is acceptable for Phase 10).

## Responsive Behavior

The start page is displayed in the browser content area (full window width minus
sidebar if open). Minimum supported content width: 640px (sidebar at 320px +
start page hero at 320px minimum).

| Viewport content width | Behavior |
|-----------------------|----------|
| >= 720px | Full 3-column suggestion card grid, favorites auto-fill at minmax(92px, 1fr) |
| 640–720px | Suggestion cards collapse to 1-column via `@media (max-width: 720px)` in styles.css |
| < 640px | Hero max-width clips at 100%; omnibox wraps; rare (sidebar would need to close first) |

The start page does not implement a mobile breakpoint — the browser is a desktop
application with a minimum window width of approximately 800px.
