## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Bash script -> source files | Smoke script reads source files via rg; script itself runs with shell execution privilege |
| release-validate.sh -> smoke-fsb-plus-parity.sh | Parent script invokes child with shell exec; exit code propagates as blocking gate |
| Parity matrix -> claimed status | Human or automation reads matrix to make release decisions; false "Better/Verified" status is a repudiation risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Repudiation | FSB-PARITY.md Better/Verified rows | mitigate | Enforce that Better rows require non-empty Native Improvement and Validation columns; smoke gate validates required categories are present and cannot be removed silently |
| T-08-01b | Tampering | smoke-fsb-plus-parity.sh static checks | mitigate | require_rg checks are source assertions that fail release if key patterns are removed; gate is blocking in release-validate.sh |
| T-08-01c | Information Disclosure | FSB-PARITY.md content | mitigate | File must not contain raw secret values, literal API keys, or passwords; matrix examples use placeholder or typed-error values only |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No new package manager installs required for this plan; stack is in-tree Bash/rg/curl only |
</threat_model>
