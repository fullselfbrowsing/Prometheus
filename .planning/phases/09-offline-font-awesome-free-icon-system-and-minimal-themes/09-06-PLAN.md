    The app icon may have been created in Phase 2 as a placeholder. This task verifies whether it is the real canonical PM mark and replaces it if not.

    Step 1 — Determine whether the existing icon is canonical or a placeholder:

    Check what the existing files contain:
    `file falkon/src/lib/data/icons/exeicons/prometheus.icns 2>/dev/null`
    `file falkon/src/lib/data/icons/exeicons/prometheus.svg 2>/dev/null`

    If prometheus.svg exists, inspect it:
    `grep -i "gradient\|ff6b35\|ff8c42\|poppins\|PM\|rounded" falkon/src/lib/data/icons/exeicons/prometheus.svg | head -20`

    A placeholder typically has: no gradient, no PM text, no orange color, or contains a generic Falkon/browser icon shape. The canonical PM mark has: orange gradient background (#ff6b35 → #ff8c42), rounded square shape with 24% corner radius, white PM text (or equivalent visual mark), no Falkon-specific content.

    Step 2 — If the existing icon IS the canonical PM mark (orange gradient, rounded square, non-Falkon):
    Document this in the task output as "LOGO-01: app icon verified as canonical PM mark — no regeneration needed." Confirm the CMakeLists.txt or Info.plist references it correctly. Skip to step 5.

    Step 3 — If the existing icon is a placeholder (or if the files are missing):
    Create the canonical PM mark SVG:

    Write `falkon/src/lib/data/icons/exeicons/prometheus.svg` with the following structure:
    - viewBox="0 0 128 128"
    - Background: a rounded rectangle (rx="30.72" which is 24% of 128) filled with linearGradient from #ff6b35 at 135deg to #ff8c42
    - Two text elements centered in the square:
      - "P" in Poppins Thin 100 (font-weight="100") in white (#ffffff), positioned so P and M appear side by side
      - "M" in Poppins Black 900 (font-weight="900") in white (#ffffff), adjacent to P
    - For the SVG, use font-family="Poppins" for both letters; the host system or rsvg will resolve the weight. Set the two text elements at the correct vertical center using dominant-baseline="middle" and text-anchor for alignment.
    - The gradient definition uses gradientTransform to achieve 135deg direction.

    The SVG source for the canonical 128px icon:

    ```svg
    <?xml version="1.0" encoding="UTF-8"?>
    <!-- Prometheus app icon — canonical PM mark -->
    <!-- Rounded square 24% corner radius, orange gradient, PM in Poppins Thin/Black -->
    <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 128 128" width="128" height="128">
      <defs>
        <linearGradient id="grad" x1="0%" y1="0%" x2="100%" y2="100%">
          <stop offset="0%" stop-color="#ff6b35"/>
          <stop offset="100%" stop-color="#ff8c42"/>
        </linearGradient>
      </defs>
      <!-- Rounded square background: 24% corner radius = 30.72px on 128px canvas -->
      <rect width="128" height="128" rx="30.72" ry="30.72" fill="url(#grad)"/>
      <!-- PM monogram: P Poppins Thin 100, M Poppins Black 900, white -->
      <text x="32" y="72" font-family="Poppins, sans-serif" font-weight="100"
            font-size="64" fill="#ffffff" text-anchor="middle"
            dominant-baseline="middle">P</text>
      <text x="96" y="72" font-family="Poppins, sans-serif" font-weight="900"
            font-size="64" fill="#ffffff" text-anchor="middle"
            dominant-baseline="middle">M</text>
    </svg>
    ```

    Step 4 — Generate .icns from the SVG:
    On macOS, use `rsvg-convert` (from librsvg) or `sips` + `iconutil` to produce the .icns:

    ```bash
    # Generate PNG exports at required sizes
    rsvg-convert -w 128 -h 128 prometheus.svg -o icon_128.png
    rsvg-convert -w 80  -h 80  prometheus.svg -o icon_80.png
    rsvg-convert -w 48  -h 48  prometheus.svg -o icon_48.png

    # Assemble icns via iconutil
    mkdir prometheus.iconset
    cp icon_128.png prometheus.iconset/icon_128x128.png
    cp icon_80.png  prometheus.iconset/icon_40x40@2x.png
    cp icon_48.png  prometheus.iconset/icon_24x24@2x.png
    iconutil -c icns prometheus.iconset -o prometheus.icns
    cp prometheus.icns falkon/src/lib/data/icons/exeicons/prometheus.icns
    ```

    If `rsvg-convert` is not available, use `qlmanage -t -s 128 -o .` or a Python script with `cairosvg`:
    ```bash
    python3 -c "import cairosvg; cairosvg.svg2png(url='prometheus.svg', write_to='icon_128.png', output_width=128, output_height=128)"
    ```

    If neither tool is available on the build machine, create the placeholder note and leave the .icns to be regenerated manually — but the SVG MUST be written so it is ready for the iconutil pipeline.

    Step 5 — Confirm CMakeLists.txt references the icon:
    `grep -n "prometheus.icns\|MACOSX_BUNDLE_ICON\|ICNS\|icns" falkon/CMakeLists.txt | head -10`

    If the reference is missing or points to a different icon, add or update:
    `set_source_files_properties(src/lib/data/icons/exeicons/prometheus.icns PROPERTIES MACOSX_PACKAGE_LOCATION "Resources")`
    and ensure `MACOSX_BUNDLE_ICON_FILE` is set to `prometheus`.
