<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| settings file -> chrome runtime | Stored user/profile settings affect native toolbar layout. |
| preferences UI -> settings file | UI selections write values consumed by all browser windows. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Tampering | `QzSettings::loadSettings` | mitigate | Clamp unknown `tabLayout` and `tabDisplay` values to compatible defaults before UI consumes them. |
| T-07-01-02 | Denial of Service | `Preferences::saveSettings` | mitigate | Save only the two documented enum strings so malformed preference state cannot break runtime layout reload. |
| T-07-01-03 | Repudiation | Preferences settings | accept | Layout/display changes are local non-sensitive preferences and existing settings persistence has no audit trail. |
</threat_model>
