    --- prometheus-common.qss ---
    Contains structural rules shared by both dark and light variants. CRITICAL: this file must contain ZERO literal accent hex values (#ff6b35, #ff8c42, etc.). All accent-colored rules use the substitution tokens @@ACCENT@@ and @@ACCENT2@@. Additionally, focus ring and accent-soft rules use @@ACCENT_SOFT2@@ which expands to rgba(R,G,B,0.18) derived from the chosen accent. The substitution method in mainapplication.cpp (Task 2) replaces all three tokens before calling setStyleSheet.

    Rules for what belongs in common vs variant files:
    - common: radii, sizing, structural layout, property-based rules whose COLOR comes from a token
    - variant: all literal surface/text/border hex values

    --- main.css ---
    The prometheus/main.css file is the default entry point that loadTheme("prometheus") reads. It contains only a brief comment; the real content is assembled at runtime by loadPrometheusVariant(). However, for the static loadTheme() path to produce a usable result before loadPrometheusVariant() is invoked, main.css should contain the dark variant content with tokens already substituted to the default FSB Orange values (#ff6b35 / #ff8c42 / rgba(255,107,53,0.18)). This means main.css is a pre-substituted snapshot of prometheus-common.qss + prometheus-dark.qss with @@ACCENT@@ replaced by #ff6b35, @@ACCENT2@@ by #ff8c42, and @@ACCENT_SOFT2@@ by rgba(255,107,53,0.18).

    Rather than duplicating all that content manually, the executor should generate main.css as follows:
    1. Read prometheus-common.qss content
    2. Read prometheus-dark.qss content
    3. Concatenate: common first, then dark variant
    4. Replace all @@ACCENT@@ with #ff6b35, @@ACCENT2@@ with #ff8c42, @@ACCENT_SOFT2@@ with rgba(255,107,53,0.18)
    5. Write the result to main.css

    This ensures `loadTheme("prometheus")` works immediately (static path), and loadPrometheusVariant() subsequently handles dynamic switching.
  </action>
