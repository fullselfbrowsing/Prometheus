<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Preferences UI -> Settings | User edits controls; saveSettings writes to disk; router reads at startup |
| Settings -> agentCap enforcement | If cap is raised through preferences, router must enforce the new cap at next start or reload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03 | Elevation of Privilege | Preferences agentCap SpinBox | mitigate | SpinBox range enforced to 1-16 at UI level; router also clamps to >0 at read time; no MCP caller can write through preferences UI |
| T-08-03b | Tampering | Vault locked rows | mitigate | vaultBoundaryCheck and vaultAutofillCheck are disabled controls that do not write to Settings; invariants cannot be silently overridden through the preferences dialog |
| T-08-03c | Information Disclosure | Preferences UI secret leak | mitigate | No QLineEdit with password echo mode added to Prometheus tab; vault/provider secrets remain in the native sidebar/control panel only |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No new package manager installs; in-tree Qt UI/C++ code only |
</threat_model>
