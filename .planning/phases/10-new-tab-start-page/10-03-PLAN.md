## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `falkon:start` HTML/JS → QWebChannel → `PrometheusStartBridge` | JS content calls C++ bridge methods |
| `PrometheusStartBridge::navigateTo` → `TabbedWebView::load` | URL string from page JS reaches native navigation |
| `PrometheusStartBridge::openAgentWithPrompt` → `AgentRuntimeSidebarWidget::seedPromptAndOpen` | Prompt string reaches agent composer text field |
| Arbitrary web page → `setupWebChannel` | Attacker page attempts to register or access `qz_prometheus_start` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-07 | Elevation of Privilege | qz_prometheus_start accessible from non-start pages | mitigate | URL guard in setupWebChannel: bridge only registered when `page->url() == QUrl("falkon:start")` OR `page->requestedUrl() == QUrl("falkon:start")` |
| T-10-08 | Tampering | navigateTo navigating to javascript: or falkon: scheme | mitigate | `navigateTo` validates `qurl.scheme()` is "http" or "https"; all other schemes silently rejected before `weView()->load()` is called |
| T-10-09 | Tampering | openAgentWithPrompt auto-submitting agent task | mitigate | `seedPromptAndOpen` only calls `setPlainText` and `setCurrentIndex(0)` — does NOT call `AgentRuntime::submitTask` or `runTask`; human must press Run Task |

| T-10-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan |
