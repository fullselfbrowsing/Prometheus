# Phase 08 - UI Design Contract

> Visual and interaction contract for the native Prometheus FSB-plus control center, settings surface, and feature parity matrix. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable; native Qt desktop surface, not React/Next/Vite |
| Component library | Qt Widgets / existing Falkon native UI (`QWidget`, `QListView`, `QStackedWidget`, `QTableView`, `QFormLayout`, `QPlainTextEdit`, `QDialog`) |
| Icon library | Existing `QIcon::fromTheme` with Prometheus resource fallbacks; do not add network-loaded assets in this phase |
| Font | Native application/system font via `QFont`; no bundled web font |

Design system source: Phase 7 native compact chrome, existing Prometheus sidebar, and current Falkon theme CSS. No `components.json`, Tailwind config, PostCSS config, or shadcn registry exists in this workspace.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge gaps, inline text/control spacing |
| sm | 8px | Default row spacing, side panel margins, stacked control spacing |
| md | 16px | Label-to-control gaps, group padding, dialog body spacing |
| lg | 24px | Section header to content separation, modal button row offset |
| xl | 32px | Standard row height, command bar height, matrix row height |
| 2xl | 48px | Header band height, paired-state summary rows |
| 3xl | 64px | Full dialog top/bottom breathing room only |

Exceptions: icon-only and compact command buttons use a 32px visual square with at least a 44px effective hit target where the platform permits; existing 30px Phase 5 buttons should be normalized to 32px when touched.

## Typography

Use exactly these four text sizes and two weights in new Phase 8 UI.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.5 |
| Label | 12px | 400 | 1.35 |
| Heading | 14px | 600 | 1.25 |
| Display | 18px | 600 | 1.2 |

`Display` is reserved for the side panel/control center title and pairing code emphasis. It is not a hero style. Logs, validation commands, pairing codes, session IDs, and audit paths use the native fixed-width font at 12px/1.35.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#090909` | Main browser/control-center background and empty space inherited from Prometheus theme |
| Secondary (30%) | `#141414` | Side panel body, status bar, section backgrounds, table viewport, inactive row surfaces |
| Accent (10%) | `#ff6b35` | Primary CTA only, focus ring, active automation badge, supervised badge, selected matrix status marker |
| Destructive | `#ff6666` | Destructive action buttons, destructive badges, blocking security/error states only |

Accent reserved for: `Run Task`, `Discover Models`, `Pair Dashboard`, `Run Parity Check`, active automation badge `A`, supervised badge `S`, focused row outline, selected matrix row marker, and Prometheus logo strokes. Do not use orange for every link, every tab, or passive metadata.

Additional status colors allowed only as bounded chips/rails: `#3a7bd5` native/default, `#2f9e7e` verified/healthy, `#9b6ade` provider/model, `#d29b36` warning/partial, `#a65b5b` blocked/manual review. Text must never rely on color alone.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Run Task |
| Empty state heading | No tasks yet |
| Empty state body | Ask Prometheus to work in this browser. Tasks will appear here with progress, cost, and final result. |
| Error state | Prometheus could not load this control surface. Refresh Diagnostics; if this repeats, run release validation. |
| Destructive confirmation | Cancel Task: Cancel this task? Progress stops now; logs remain available. |

Required CTA labels: `Run Task`, `Cancel Task`, `Save Provider`, `Discover Models`, `Save Fallbacks`, `Save Permissions`, `Save Memory`, `Save Site Guide`, `Save Vault Entry`, `Pair Dashboard`, `End Pairing`, `Open Parity Matrix`, `Run Parity Check`, `Refresh Diagnostics`, `Export Diagnostics`.

Required empty states:

| Surface | Heading | Body |
|---------|---------|------|
| Tasks | No tasks yet | Ask Prometheus to work in this browser. Tasks will appear here with progress, cost, and final result. |
| Providers | No provider configured | Choose a provider, model, endpoint, and save from this native panel. |
| Model discovery | No models discovered | Add provider credentials or choose Local, then run Discover Models. |
| MCP status | MCP bridge offline | Start Prometheus with the native agent server enabled, then refresh diagnostics. |
| Permissions | No permission rules yet | Add a rule before agents can mutate internal surfaces. |
| Vault | No vault entries | Save metadata here. Secrets stay in native secure storage. |
| Memory | No memory saved | Save a note from the current page. |
| Site guides | No site guides | Save instructions for this origin. |
| Logs | No runtime logs yet | Run a task or refresh diagnostics. |
| Diagnostics | No diagnostics captured | Run Refresh Diagnostics to inspect browser, MCP, runtime, adapters, and supervision. |
| Supervision | No active supervision session | Pair a dashboard to begin supervised DOM-native viewing. |
| Parity matrix | No parity rows loaded | Generate `FSB-PARITY.md` before claiming FSB parity. |

Required error/security copy:

| Condition | Copy |
|-----------|------|
| Provider secret over transport | Provider secret blocked. Store keys from the native Prometheus UI. |
| Vault secret over transport | Vault secret blocked. Secrets cannot cross MCP or remote transports. |
| Vault autofill over transport | Vault autofill requires confirmation. Bring the tab forward and approve in Prometheus. |
| Agent cap reached | Agent cap reached. End an active session or raise the cap in Settings. |
| Tab owned by another agent | Tab owned by another agent. Switch to an owned tab or release ownership. |
| Supervision expired | Supervision session expired. Pair the dashboard again. |
| Supervision target mismatch | Supervision target mismatch. The session belongs to a different tab. |
| Permission blocked | Permission blocked. Enable internal-surface control for this scope. |
| Model discovery failed | Model discovery failed. Check provider key, endpoint, and network, then run Discover Models. |

Required destructive confirmations:

| Action | Confirmation copy |
|--------|-------------------|
| Cancel Task | Cancel this task? Progress stops now; logs remain available. |
| End Pairing | End this supervision session? The dashboard will stop receiving snapshots for this tab. |
| Clear Runtime Logs | Clear runtime logs? This removes local log history from this profile. |
| Delete Vault Entry | Delete vault entry metadata for `%1`? Stored secret access will be removed from Prometheus. |
| Remove Provider Key | Remove saved key for `%1`? Tasks using this provider will fall back to the next enabled provider. |
| Reset Permission Rule | Reset permissions for `%1`? Agents will lose access until permission is granted again. |
| Unpair Dashboard | Unpair this dashboard? Remote control actions from this session will be rejected. |

## Native Layout Contract

### Control Center Shell

- Use the existing `PrometheusAgent` sidebar entry and `Ctrl+Shift+P` shortcut.
- Root object remains `PrometheusAgentSidebar`.
- Minimum width: 320px. Preferred width: 384px. Wide dialog/control-center mode starts at 720px.
- Root margins: 8px. Root spacing: 8px. No decorative backgrounds, gradients, marketing hero, or card-heavy layout.
- Header height: 48px. Title `Prometheus` uses Display typography; tagline `Powered by FSB` uses Label typography.
- Header status line shows compact text chips: `MCP: %1`, `Agents: %1/%2`, `Vault: Native-only`, `Supervision: %1`.
- Do not use a horizontal `QTabWidget` for all Phase 8 categories. Use a section selector plus `QStackedWidget`.
- At widths under 420px, the section selector is a `QComboBox` above the current page. At 420px and wider, it may become a 120px `QListView` section rail beside the stacked detail page.

Required sections, in order:

1. `Tasks`
2. `Providers & Models`
3. `MCP Status`
4. `Permissions & Agents`
5. `Vault`
6. `Memory & Site Guides`
7. `Logs & Diagnostics`
8. `Supervision & Pairing`
9. `Parity Matrix`

### Settings Rows

- Standard row height is 32px. Optional helper text adds one 16px line below the row without changing sibling row heights.
- Side panel label column: 136px. Wide dialog label column: 180px.
- Controls use native widgets: `QCheckBox` for binary settings, `QSpinBox` for agent caps, `QComboBox` for permission modes/providers/models, `QLineEdit` for endpoint/model/search, `QPlainTextEdit` only for prompts/guides/log details.
- Every setting row has a stable accessible name matching its visible label.
- Required settings rows: `Internal surface control`, `Agent cap`, `Tab ownership enforcement`, `Background tab actions`, `Visual feedback`, `Telemetry`, `Vault secret transport`, `Vault autofill confirmation`, `Supervision pairing`, `Dashboard remote control`.
- Locked security rows use disabled controls plus the visible value `Locked on`; do not hide locked policies.

### Tasks

- Prompt editor height: 96px. It does not grow while typing.
- Command row height: 32px with `Run Task` as the only accent-filled button and `Cancel Task` as destructive only while a cancellable task is selected.
- Task list uses model-backed 32px rows. Row text order: status, prompt/title, provider/model, latest progress, cost/action counts.
- Status badges are text labels, not just dots.
- Selecting a task opens a detail area with `Progress`, `Final result`, `Metrics`, `Logs`, and `Cancel Task` if applicable.
- Provider-backed, local/offline, and MCP-driven execution paths must be visible in the selected task details as `Execution: Provider`, `Execution: Local/offline`, or `Execution: MCP`.

### Providers & Models

- Provider list rows are 32px high and include provider name, enabled/fallback status, key state, and discovery state.
- Supported provider labels: `Hosted`, `Routed`, `Local`, `OpenAI`, `Anthropic`, `Gemini`, `xAI`, `OpenRouter`, `LM Studio`, `Custom`.
- Provider detail includes `Enabled`, `Model`, `Endpoint`, `Key`, `Discover Models`, and fallback order.
- Secret fields use password echo mode and clear after save. Never show saved secret values; show only `Key saved` or `No key saved`.
- Model discovery result rows use 32px rows with model name, source provider, context/capability notes when known, and fallback eligibility.
- Fallback order is a model-backed list with drag/drop or up/down buttons. Empty fallback order shows `No fallback order set`.

### MCP Status

- Status panel covers native server health, authorization mode, bridge transport, audit path, tool count, and adapter health.
- Use a compact two-column key/value table with 32px rows.
- Tool coverage list must include manual actions, read tools, runtime tools, provider tools, vault tools, memory/site-guide tools, supervision tools, and compatibility stubs.
- Unauthorized or offline states use the exact error/security copy above and include `Refresh Diagnostics`.

### Permissions & Agents

- Permission matrix uses rows for `Preferences`, `Prometheus control center`, `Downloads`, `History`, `Vault`, `Provider settings`, `Supervision`, and `Page JavaScript`.
- Columns: `Surface`, `Mode`, `Agent scope`, `Audit`, `Last changed`.
- Allowed modes: `Allowed`, `Ask every time`, `Blocked`. Default is `Ask every time` for internal-surface mutations and `Blocked` for vault secret transport.
- Agent caps use a `QSpinBox` range 1-16 with default 4, matching current router behavior unless persisted settings override it.
- Tab ownership setting defaults to enabled and must display current owner counts from diagnostics/list tabs.
- Visual feedback setting controls visible status/badge behavior but does not suppress audit logging.
- Telemetry copy must distinguish local runtime logs from any future remote telemetry. Default visible value: `Local only`.

### Vault

- Vault list rows are 32px high: origin, label, secret state, autofill policy.
- Secret boundaries are always visible: `Native-only`, `Remote blocked`, and `Confirmation required`.
- Secret creation/editing happens only in native foreground UI.
- Autofill action requires foreground confirmation and must show origin, label, target tab, owner, supervision state, and destination field before approval.
- No vault row, log row, tooltip, diagnostic payload, or parity evidence can include raw secret values.

### Memory & Site Guides

- Use two stacked native sections, not nested cards.
- Memory row order: key, current URL/origin, updated timestamp, note preview.
- Site guide row order: origin, updated timestamp, instruction preview.
- Editors use fixed heights: memory note 88px, site guide instructions 112px.
- Save buttons are normal native buttons; use accent only when the edited field is dirty and valid.

### Logs & Diagnostics

- Logs use `QPlainTextEdit` or model-backed list plus detail pane with fixed-width 12px text.
- Log filters: `All`, `Task`, `Provider`, `Memory`, `Vault`, `Supervision`, `Error`.
- Diagnostic groups: `Browser`, `Native server`, `MCP bridge`, `Adapters`, `Agents`, `Ownership`, `Runtime`, `Supervision`, `Release validation`.
- Command/audit paths are selectable and elided in the middle when they exceed available width.
- Diagnostics errors must preserve typed error codes such as `SECRET_TRANSPORT_BLOCKED`, `VAULT_NATIVE_CONFIRMATION_REQUIRED`, `AGENT_CAP_REACHED`, `TAB_OWNED_BY_OTHER_AGENT`, `STALE_SUPERVISION_SESSION`, and `SUPERVISION_TARGET_MISMATCH`.

### Supervision & Pairing

- Pairing summary row height: 48px.
- Pairing code is 8 characters, uppercase, fixed-width, 18px/600.
- Required fields: pairing code, session ID, target tab, target key, sequence, expires at, active agent, snapshot hash, last seen.
- `Pair Dashboard` is accent. `End Pairing` and `Unpair Dashboard` are destructive.
- Expired/stale/target-mismatch sessions remain visible until dismissed and use the exact security copy above.
- Active supervision state must also be reflected in compact tab/search badges as `S` with tooltip `Supervised session active`.

### Parity Matrix

- Primary artifact access is `Open Parity Matrix`; it opens or focuses `FSB-PARITY.md`/native parity view.
- Side panel condensed matrix columns: `Category`, `Native status`, `Improvement`, `Evidence`.
- Wide matrix columns: `Category`, `FSB baseline`, `Native status`, `Improvement target`, `Validation command`, `Evidence`, `Owner/notes`.
- Row height: 32px. Header height: 28px. Status chips are bounded to 92px and use labels from the status table.
- Required categories: side panel, provider settings, model discovery, fallback routing, MCP tools, DOM snapshots, page snapshots, read tools, action tools, smart waiting, stuck detection, visual feedback, memory, site guides, vault, remote dashboard pairing, supervision DOM mirror, tab ownership, tab groups/state, diagnostics, logs/action history, cost/token accounting, telemetry controls, internal settings control, release validation.
- `Better` status requires a native improvement target and a validation command. It cannot be set from copy alone.
- `Verified` status requires an evidence link/path and a passing validation command.

## Interaction Contract

- All section changes preserve the current section's unsaved edits and show dirty state in the section selector.
- Saving a provider, permission, memory item, site guide, or vault entry updates the relevant list immediately and writes a runtime log.
- Destructive actions require a native confirmation dialog with the exact confirmation copy and any tab owner/supervision state summary shown as disabled rows.
- Background refresh must not steal browser focus. Foreground confirmation actions may raise the relevant native dialog.
- Agent-visible settings must remain auditable: changes write runtime logs or audit entries with timestamp, surface, old mode when available, new mode, and actor/source.
- Internal-surface permission changes must not be controllable by MCP unless explicitly allowed by a native setting and visible in the permission matrix.
- Secret transport blocking is a product invariant. Provider and vault secret values are native UI only.

---

## Accessibility and Keyboard

- Every control has visible text or an accessible name. Icon-only buttons must have tooltips.
- Color is never the only status indicator; badges and chips include text or accessible descriptions.
- Tab order follows section selector, page title/action row, primary form/list, detail pane, destructive actions.
- Keyboard shortcuts: `Ctrl+Shift+P` opens Prometheus, `Ctrl+F` focuses the current section filter, `Ctrl+Enter` runs the task prompt, `Esc` closes popovers or cancels pending confirmation, `Space` toggles checkboxes, `Enter` activates the focused primary button.
- Focus ring uses the accent color at 1px, matching Phase 7 delegates.
- Logs, diagnostics, audit paths, validation commands, and pairing/session IDs are selectable text.
- Minimum contrast target is 4.5:1 for normal text and 3:1 for large or non-text indicators against the active palette.

---

## Responsive and Stability Rules

- Minimum functional side panel width is 320px. At this width, all pages are single-column and optional columns collapse into details.
- At 420px, a section rail may appear beside the detail page.
- At 720px or wider, the same content may open as a two-pane control-center dialog with section rail, detail page, and optional inspector/detail pane.
- Text in rows is elided, not allowed to resize rows unexpectedly. URLs, commands, and paths use middle elision.
- Fixed-format elements must declare stable dimensions: 32px rows, 48px summaries, 96px prompt editor, 88px memory editor, 112px site guide editor.
- No visible text may overlap adjacent buttons, badges, or status chips at 320px, 384px, 480px, and 720px widths.

## Verification Checklist

- Native side panel opens with `Ctrl+Shift+P` and exposes all nine required sections.
- Width checks pass at 320px, 384px, 480px, and 720px with no overlapping text and no layout jump from dynamic content.
- Task flow covers provider-backed configured mode, local/offline fallback, MCP submission, cancellation, final result, progress, and error states.
- Provider flow covers all required providers, model discovery empty/success/failure, fallback ordering, and native-only secret save.
- MCP/diagnostics view shows server, bridge, adapters, authorization, audit path, ownership, runtime, and supervision status.
- Permission settings expose internal-surface permissions, agent cap, tab ownership, background actions, visual feedback, telemetry, vault boundaries, and supervision pairing state.
- Vault flow proves metadata can cross native/runtime surfaces while raw secrets cannot cross MCP or remote transports.
- Memory and site guides save, list, and display empty/error states.
- Supervision pairing shows active, expired, stale, target mismatch, and ended states with exact copy.
- Parity matrix includes every required category and prevents `Better`/`Verified` without improvement target, validation command, and evidence.
- Release validation includes a parity gate and exercises side panel, MCP bridge, browser-owned settings, provider config, memory/site guides, vault, diagnostics, supervision, and release packaging.
- Static checks pass: `node --check falkon/tools/prometheus-mcp/server.mjs`, `bash -n falkon/tools/fsb-baseline/smoke-agent-control.sh falkon/tools/fsb-baseline/release-validate.sh`, and the future parity smoke script.
